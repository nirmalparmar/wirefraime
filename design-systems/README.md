# Brand design systems

Each folder under `design-systems/<id>/` is a committed **brand package** the
user can pick at create time (the "Brand" axis — see
`docs/architecture-plan.md`). Picking one makes generation adopt that brand's
colors, fonts, and layout feel; the default is "Custom (AI-generated)".

## Package shape

```
design-systems/<id>/
├── manifest.json         # picker metadata (see _schema/manifest.schema.ts)
├── design-system.json    # renderable Wirefraime DesignSystem (colors/fonts/layout)
└── DESIGN.md             # brand direction prose, injected as generation context
```

`design-system.json` uses Wirefraime's own token contract (colors.primary →
`--color-primary` → `bg-primary`, …), so the live click-to-edit editor and
iframe rendering work unchanged. DESIGN.md is **direction only** (mood,
proportions, anti-patterns) — the loader frames it so the model uses tokens,
not the literal hex/font names in the prose.

## Loader

`lib/design/design-systems.ts`:
- `listBrandPackages()` — catalog metadata for the picker (`/api/design-systems`).
- `loadBrandPackage(id)` — `{ manifest, designSystem, brandContext }`.
- `mergeBrandIntoDesignSystem(ai, brand)` — brand owns colors/fonts/visual feel;
  the AI plan keeps the app-specific nav style + structure.

## Adding / re-syncing

Packages are generated by `scripts/import-design-systems.ts` from Open Design's
catalog. Edit the `SEEDS` table there (slug, category, blurb, Google font) and
re-run `bun scripts/import-design-systems.ts`. The sibling `../open-design` repo
is only needed to (re)generate; the emitted packages are self-contained and ship
on their own.
